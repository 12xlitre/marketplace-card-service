# Схема авторизации и сессий

Дата: 2026-07-08.

Фиксированное решение после нескольких итераций:
- аутентификация в продакшн-пути идет только через backend `/api/login`;
- `/api/session` является единственным источником восстановления сессии после `refresh`;
- пароли не кешируются в браузере и не хранятся в frontend/CSV;
- список сотрудников для селектов берется из `/api/users`, при отсутствии backend-ответа — из локального `document/test-users.csv` как fallback.

Проектные правила:
- remember-me реализован через тот же куки-сессионный механизм:
  - без `remember`: TTL `12` часов,
  - с `remember`: TTL `7` дней;
- сервер выставляет `HttpOnly` cookie, `SameSite=Lax`, и контролирует истечение на серверной стороне;
- кнопка `Выйти` очищает сессию и cookie.

Ограничения:
- текущая интеграция с WB/карточками пока read-only;
- write-действия в WB (обновления карточек, цены, остатки, публикация) запрещены на этом этапе.
