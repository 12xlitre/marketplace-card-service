# Reports

Дата: 2026-07-13.

Устойчивое продуктовое решение:
- отчеты должны жить во вкладке `Отчеты` внутри каждого кабинета селлера, а не в глобальном левом меню и не внутри аудита отдельной карточки;
- первый отчет в разделе - клиентский WB XLSX по кабинету: контент, цены, остатки, заказы, продажи, реклама, акции и доступность источников;
- будущие отчеты добавляются в этот же кабинетный каталог отдельными карточками/типами отчетов;
- рабочий сценарий: пользователь заходит в кабинет селлера -> вкладка `Отчеты` -> выбирает тип отчета -> выбирает период через `дата начала / дата конца` -> нажимает `Сформировать`;
- после формирования файл скачивается, а в кабинете появляется запись истории отчета с типом, периодом, датой, именем файла и возможностью сформировать снова;
- не зашивать период из клиентского примера в UI или backend-логику;
- backend принимает явный период `start/end` для `GET /api/portals/{id}/wb-client-report`, а `weeks` остается только fallback совместимости;
- отчет остается read-only: WB API используется только для чтения, без write-операций.
