# Reports

Дата: 2026-07-13.

Устойчивое продуктовое решение:
- отчеты должны жить во вкладке `Отчеты` внутри каждого кабинета селлера, а не в глобальном левом меню и не внутри аудита отдельной карточки;
- первый отчет в разделе - клиентский WB XLSX по кабинету: основной рабочий лист периода плюс служебные листы `WB_Воронка_текущая`, `WB_Воронка_прошлая`, `WB_Цены_загрузка`, `WB_РК_API`, `WB_Акции_загрузка`, `WB_Акции_план`, `Остатки`, `Источники`;
- основной лист близок к ручному клиентскому формату специалиста: воронка, реклама, акции, остаток в днях, локальность, динамика заказов, автоматическая подсказка `Предложение ВебОптимайз` и ручные поля для планов/комментариев/отметки выполнения;
- в Ozon beta вкладка `Отчеты` тоже показывает кабинетный каталог, но первый отчет там отдельный: `Ozon клиентский XLSX`, собран из Ozon snapshot/MPStats без WB API;
- Ozon XLSX повторяет клиентский Ozon-референс: основной лист периода и служебные листы `OZON_СВОД_ПО_SKU`, `OZON_АНАЛИТИКА_ЗАГРУЗКА`, `OZON_ЦЕНЫ_ОСТАТКИ`, `OZON_CPO_ЗАГРУЗКА`, `OZON_ОБОРАЧИВАЕМОСТЬ`, `OZON_ПЛАТНОСТЬ`, `OZON_ЛОКАЛЬНОСТЬ`, `OZON_ЗАКАЗЫ_ЗАГРУЗКА`, `OZON_ТОВАРЫ_ЗАГРУЗКА`, `OZON_СТОК_ЗАГРУЗКА`, `OZON_ФИНАНСЫ_ЗАГРУЗКА`, `OZON_ПОКРЫТИЕ_ДАННЫХ`, `OZON_ИСТОЧНИКИ`;
- пока Ozon Seller API не подключен, отчет обязан честно показывать покрытие данных: что уже заполнено из Ozon snapshot/MPStats, что заполнено частично, а какие коммерческие блоки ждут API или ручной загрузки;
- будущие отчеты добавляются в этот же кабинетный каталог отдельными карточками/типами отчетов;
- рабочий сценарий: пользователь заходит в кабинет селлера -> вкладка `Отчеты` -> выбирает тип отчета -> выбирает период через `дата начала / дата конца` -> нажимает `Сформировать`;
- после формирования файл скачивается, а в кабинете появляется запись истории отчета с типом, периодом, датой, именем файла и возможностью сформировать снова;
- не зашивать период из клиентского примера в UI или backend-логику;
- backend принимает явный период `start/end` для `GET /api/portals/{id}/wb-client-report`, а `weeks` остается только fallback совместимости;
- отчет остается read-only: WB API используется только для чтения, без write-операций;
- этот раздел не является `Отчетным периодом` отдела: клиентские XLSX-отчеты показывают данные WB за выбранные даты, а отчетный период отдела хранит план работ менеджера/техспециалиста по кабинету.
