# Semantic Core

Дата: 2026-07-14.

Устойчивое решение:
- вход во вкладку `Семантическое ядро` сначала показывает стабильные ранжирующиеся запросы карточки из MPStats `/analytics/v1/wb/items/{nmID}/keywords`; смена стартового SEO-запроса не должна менять этот набор, а должна менять только кандидатов к добавлению;
- `Ключи в карточке` не должны определяться мягким совпадением отдельных слов по всему описанию: query считается уже заложенным в контент только при строгом фразовом совпадении или короткой двухсловной перестановке;
- SEO expansion MPStats должен сохранять полный рабочий набор `allKeywords` до 5000 строк, иначе высокочастотные, но не первые в текущем срезе запросы исчезают из правой колонки;
- стартовый запрос MPStats можно расширять дополнительными seed-фразами из самой карточки, если они явно читаются в названии, описании или характеристиках; пример: для описания `очки кошки` добавляем `очки кошачий глаз`, чтобы получить cat-eye запросы;
- оператор может заменить стартовый запрос в поле СЯ и нажать Enter: это запускает новый подбор по введенной фразе и меняет только кандидатов к добавлению; для карточек очков ручной seed без слова `очки`, например `кошачий глаз`, дополнительно квалифицируется товарными seed-фразами `очки ...` / `солнцезащитные очки ...`;
- архивные подборки СЯ являются отдельной кабинетной библиотекой: применение копирует ключи в карточку, пополнение добавляет в архив только новые ключи из карточки, а удаление из архива не меняет уже сохраненные карточки;
- во вкладке `Изменения -> Контент` подсветка ключей в черновике описания должна быть отдельным превью рядом с textarea: специалист видит связь фрагмента текста с ключом, но редактирование остается в обычном поле без потери курсора.
- дополнительные seed-фразы требуют простого товарного guard: для карточек про очки кандидаты должны содержать корень `очк`, чтобы не протаскивать нерелевантные строки вроде `черная кошка`;
- частотные корзины по умолчанию: высокий спрос `>=1000`, средний `>=300`, низкий `<300`; backend-пороги задаются `MPSTATS_SEMANTIC_HIGH_FREQUENCY` и `MPSTATS_SEMANTIC_MEDIUM_FREQUENCY`, автодобавление берет максимум 36 запросов, до 12 из каждой корзины.
- действующие `Ключи в карточке` можно пометить `к удалению`: это предложение для согласования, а не автоматическая правка WB. Итоговый отчет СЯ показывает такие ключи отдельными колонками `Ключ к удалению из карточки`, `Причина удаления`, `Согласование удаления`; после согласования специалист вручную удаляет ключ в SEO карточки WB и проводит переоптимизацию без него.
