# Approval Workflow

Дата фиксации: 2026-07-08.

Роли процесса:
- технический специалист готовит черновик изменений;
- аккаунт-менеджер проекта согласовывает: `Принято` или `На доработку` с обязательной причиной;
- руководитель отдела наблюдает процесс и статистику, но не редактирует карточку и не согласует вместо аккаунт-менеджера;
- админ может технически выполнить согласование.

Хранение:
- текущий статус задачи живет в `card_drafts.meta.approval`;
- переходы статусов пишутся в `card_approval_events` для аналитики и не удаляются при сбросе черновика;
- `GET /api/approval-workflow?portal_id=...` отдает текущие задачи, историю и простую аналитику по кабинету.

Обновление WB:
- обычная кнопка `Загрузить свежие данные` не удаляет черновик и задачу согласования;
- обновление WB выполняет soft reset анализа через `POST /api/portals/{id}/reset-analysis-cache`;
- soft reset очищает MPStats-кэш и помечает audit-черновики как `stale`, но сохраняет значения в колонке `Стало` и `meta.approval`.
