# Work Periods

Дата: 2026-07-14.

Устойчивое продуктовое решение:
- `Отчетный период` - это вкладка внутри конкретного кабинета селлера с планом работ отдела, а не клиентская XLSX-выгрузка из вкладки `Отчеты` и не MPStats-период;
- период создается карточкой/квадратом по сценарию, похожему на создание кабинета: пользователь нажимает `Создать`, задает название и произвольные даты начала/конца через календарь;
- даты не привязаны к календарному месяцу, потому что работа может начаться в любой день;
- при создании пользователь выбирает чек-лист направлений работ; базовый список: `Семантика`, `Контент`, `Цены`, `Остатки`;
- каждый пункт плана хранит статус `planned/done/returned`, комментарий, дату выполнения, исполнителя, причину возврата, дату возврата и историю действий;
- выполнение фиксируется явным действием менеджера/специалиста: при отметке `выполнено` сохраняются комментарий и дата;
- возврат обязателен с причиной, старые данные выполнения не стираются без истории;
- в конце периода пользователь формирует отчет: сколько пунктов выполнено, что не выполнено или возвращено, и почему;
- данные хранятся в backend-таблице `portal_work_periods`, строго в рамках `portal_id`;
- маршрут `GET/POST/DELETE /api/portal-work-periods` обязан проверять доступ пользователя к кабинету через `user_can_access_portal`.
